# UX Guidelines

This folder contains UX guidelines for the dadamachines TBD‑16 (CTAG TBD fork).

Purpose
- Provide clear, practical guidance for UI/UX decisions across the project.
- Ensure consistent design patterns, terminology, and accessibility standards.

How to use
- Add design rationale, examples, and dos/don'ts for components and flows.
- Link to concrete code or storybook components when available.

Suggested sections
- Principles — design values and goals for the product.
- Terminology — preferred product and UI wording.
- Layout & spacing — grid, margins, and alignment rules.
- Components — buttons, inputs, lists, and controls (usage and states).
- Patterns — navigation, dialogs, forms, error handling.
- Visuals — typography, color palette, iconography guidelines.
- Accessibility — keyboard, screen reader, color contrast, and motion.
- Contribution — how to propose and document UX changes (PR checklist).

Add more content here as the team's guidelines evolve.
