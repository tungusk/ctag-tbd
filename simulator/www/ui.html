About this tab — the legacy ctag-tbd plugins (CV/Gate/Pot, no MIDI)
The TBD's hardware modulation inputs: 4 CV inputs, 2 trigger/gate inputs, and the 2 front-panel pots. All the original ctag-tbd plugins are Eurorack-style and respond only to these — they have no MIDI input. You wire which control drives which plugin parameter from the main WebUI (the small dropdown next to each parameter), exactly like on the hardware module. So: load a plugin, map a parameter to CV0 / TRIG0 / POT0…, then drive it from here. (The TBD-16's GrooveBoxRack is the exception — see the other tab.)

About this tab — GrooveBoxRack is the TBD-16's one MIDI plugin; how MIDI maps to its 16 tracks
Only GrooveBoxRack responds to MIDI — the TBD-16 is a MIDI device (not Eurorack), and this rack hosts up to 16 “machines” (drum voices + synths). Load GrooveBoxRack on a channel in the main WebUI, then play it from here. Heads-up: the rack's MIDI channels don't line up with the WebUI's CHnn channel-strip numbers — the 8 drum tracks (CH01–08) are on MIDI ch 10/11/12 (the note 36/37/38 picks the track), the synth tracks (CH09–15) on MIDI ch 1–7 (one per track), and the audio-input track (CH16) on MIDI ch 8; MIDI ch 9 + 13–16 are unused. The canonical map is data/synthdefinitions.json.

Plug a USB MIDI controller into your machine, click Enable WebMIDI, pick the input, and play. Notes, velocity and CC all route through to the track selected in the Channel dropdown below — the controller's own MIDI channel is ignored, so you don't have to reconfigure it. Change the dropdown to switch tracks, or change the machine on the WebUI's track strip — the hardware always plays whatever is currently loaded on the selected track. CC values reach the rack's handleMidiControlChange on the same channel, so knobs and the mod wheel move whichever params the active machine has registered for those CCs.
